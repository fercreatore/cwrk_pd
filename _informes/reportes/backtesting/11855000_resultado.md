## Propuesta de mejora

### 1. Desestacionalizar vel_real SIEMPRE

El error principal es usar vel_real cruda (117/mes) cuando proviene de meses pico. La vel_real debe dividirse por el factor estacional del mes para obtener la velocidad base.

### 2. Aplicar tendencia inter-anual

La familia mostro:
- Pico: 1009 pares OI2021
- Caida: -35% (2022), -39% (2023), -8% (2024)
- Rebote: +41% (2025 = 516 pares)

Un promedio movil de 3 anios suaviza la volatilidad y captura mejor la tendencia.

### 3. Timing: pedir con lead_time + colchon

Regla: `fecha_pedido = inicio_OI (1-abr) - lead_time (60 dias) - colchon (15 dias)`.
Para OI: pedir en **primera quincena de enero**.

### 4. Cap de compra: no superar 1.2x demanda proyectada

Para evitar sobrestock, limitar la compra a 120% de la demanda neta (demanda - stock_inicio).

### Codigo Python del ajuste recomendado

```python
def proyectar_demanda_oi(vel_real_por_mes, factores_mes_origen, factores_estacionales,
                         meses_objetivo=None):
    """
    Proyecta demanda OI desestacionalizando la vel_real.

    Args:
        vel_real_por_mes: dict {(anio, mes): ventas} de meses NO quebrados
        factores_mes_origen: dict {mes_int: factor_estacional} del mes donde se midio vel_real
        factores_estacionales: dict {mes_int: factor_estacional} para los 12 meses
        meses_objetivo: lista de meses a proyectar, default [4,5,6,7,8,9] (OI)

    Returns:
        dict {mes: demanda_proyectada}, total
    """
    if meses_objetivo is None:
        meses_objetivo = [4, 5, 6, 7, 8, 9]

    # Paso 1: desestacionalizar cada observacion
    vel_base_observaciones = []
    for (anio, mes), ventas in vel_real_por_mes.items():
        factor = factores_estacionales.get(mes, 1.0)
        if factor > 0.05:  # ignorar meses con factor ~0 (no informativos)
            vel_base_observaciones.append(ventas / factor)

    if not vel_base_observaciones:
        return {m: 0 for m in meses_objetivo}, 0

    # Paso 2: velocidad base = promedio de observaciones desestacionalizadas
    vel_base = sum(vel_base_observaciones) / len(vel_base_observaciones)

    # Paso 3: proyectar cada mes objetivo
    proyeccion = {}
    for mes in meses_objetivo:
        factor = factores_estacionales.get(mes, 1.0)
        proyeccion[mes] = round(vel_base * factor, 1)

    total = sum(proyeccion.values())
    return proyeccion, round(total)


def calcular_compra_optima(demanda_proyectada, stock_inicio, cap_factor=1.2):
    """
    Calcula cantidad optima a comprar con cap de seguridad.

    Args:
        demanda_proyectada: pares proyectados para la temporada
        stock_inicio: stock disponible al inicio de temporada
        cap_factor: multiplicador maximo sobre demanda neta (default 1.2)

    Returns:
        cantidad_a_comprar (int)
    """
    demanda_neta = max(0, demanda_proyectada - stock_inicio)
    cantidad = min(demanda_neta * cap_factor, demanda_neta + 50)  # cap o +50, el menor
    return int(round(cantidad))


def fecha_pedido_optima(inicio_temporada_mes=4, lead_time_dias=60, colchon_dias=15):
    """
    Calcula fecha ideal de colocacion del pedido.

    Para OI con lead_time 60 dias: pedir en primera quincena de enero.
    Para PV con lead_time 60 dias: pedir en primera quincena de julio.
    """
    from datetime import date
    from dateutil.relativedelta import relativedelta

    hoy = date.today()
    anio = hoy.year if hoy.month < inicio_temporada_mes else hoy.year + 1
    inicio = date(anio, inicio_temporada_mes, 1)
    fecha_pedido = inicio - timedelta(days=lead_time_dias + colchon_dias)
    return fecha_pedido
```
